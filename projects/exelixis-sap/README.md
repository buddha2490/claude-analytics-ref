# Cohort: NPM-008 / Exelixis XB010-100 NSCLC ECA Study

This directory contains a complete, AI-generated clinical programming pipeline for the NPM-008 study (Exelixis XB010-100 External Control Arm), demonstrating how Claude Code can autonomously create SDTM simulation programs and derive ADaM analysis datasets.

---

## Getting Started

Follow these steps in order to generate and explore the study data.

### Step 1: Generate SDTM Data

Run the SDTM master orchestrator from the `exelixis-sap/` project root. This sources all 19 domain simulation programs in dependency order and writes XPT files to `output-data/sdtm/`.

```r
# From projects/exelixis-sap/
source("programs/sdtm/run_all_sim_sdtm.R")
```

### Step 2: Generate ADaM Datasets

Once SDTM data exists, run the ADaM orchestrator. This derives all 6 ADaM datasets in dependency order and writes XPT files to `output-data/adam/`.

```r
# From projects/exelixis-sap/
source("programs/adam/run_all_adam.R")
```

### Step 3: Explore the Data

**Option A — Interactive exploration**

Load all SDTM and ADaM datasets into named lists for interactive use in your R session:

```r
source("programs/explore-data.R")

# Access datasets by name:
sdtm_list$dm    # Demographics
sdtm_list$ae    # Adverse Events
sdtm_list$rs    # Response
# ... (19 SDTM domains)

adam_list$adsl  # Subject-Level
adam_list$adae  # Adverse Events
adam_list$adrs  # Response
# ... (6 ADaM datasets)
```

**Option B — Variable-level QA report**

Render the Quarto report to generate summary statistics for every variable across all SDTM and ADaM datasets:

```r
quarto::quarto_render("output-reports/exelixis-dummy-data-report.qmd")
# Output: output-reports/exelixis-dummy-data-report.html
```

## Plans: AI Agent Instructions

Two comprehensive planning documents guide the AI workflow:

### 1. `plan_sim_sdtm_2026-03-25.md`
**Purpose:** Generate simulated SDTM data for 40 NSCLC patients

**Approach:**
- 19 SDTM domains (DM, EX, EC, CM, PR, SU, AE, DS, HO, MH, IE, BS, LB, VS, QS, RS, TR, TU, SC)
- Dependency-aware execution order (DM → IE → MH → ... → DS)
- Reproducible simulation (seed management per domain)
- Realistic clinical trajectories:
  - BOR distribution: 18% PR, 40% SD, 35% PD, 7% NE
  - PFS: Weibull/Exponential distributions by response group
  - OS: Weibull(shape=1.2, scale=450) targeting ~12-month median
- Date de-identification with per-patient shift

**Output:** 19 XPT files in `output-data/sdtm/`

### 2. `plan_adam_automation_2026-03-27.md`
**Purpose:** Derive ADaM datasets from SDTM using multi-agent workflow

**Approach:**
- Wave-based parallel execution (4 waves, 6 datasets)
- Three-agent architecture:
  - **feature-planner:** Reviewed specs, pre-resolved open questions
  - **r-clinical-programmer:** Implemented derivations (8-step workflow)
  - **clinical-code-reviewer:** Independent QC with fix cycles
- Dependency management:
  - Wave 1 (parallel): ADLOT + ADBS
  - Wave 2: ADSL (depends on ADLOT)
  - Wave 3 (parallel): ADRS + ADAE (depend on ADSL)
  - Wave 4: ADTTE (depends on ADSL + ADRS)
- Between-wave validation checks

**Output:** 6 XPT files in `output-data/adam/`

## How AI Generated This Data

### Phase 1: SDTM Simulation

**Agent:** r-clinical-programmer
**Plan:** `plan_sim_sdtm_2026-03-25.md`

1. **Outcome assignment:** Each subject assigned latent BOR, PFS, OS based on clinical distributions
2. **Domain-by-domain simulation:** 19 programs executed in dependency order via `run_all_sim_sdtm.R`
3. **Cross-domain consistency:** All dates/events aligned to subject-level outcomes
4. **Realism features:**
   - Biomarker prevalence (EGFR 20%, KRAS 15%, ALK 2%)
   - Adverse event trajectories (treatment-emergent, severity distributions)
   - Response patterns (responders have longer PFS than progressors)
   - Date de-identification (per-patient random shift ±14 days)

**Result:** 19 CDISC-compliant SDTM XPT files

### Phase 2: ADaM Derivation

**Agents:** feature-planner → r-clinical-programmer → clinical-code-reviewer
**Plan:** `plan_adam_automation_2026-03-27.md`

**Wave-based execution:**

```
Wave 1 [parallel]:  ADLOT + ADBS
    ├─ ADLOT: NPM LoT algorithm implemented (45-day window, 120-day gap rule)
    └─ ADBS: Direct mapping from BS domain

Wave 2 [serial]:    ADSL (depends on ADLOT)
    └─ ADSL: 65 variables from 11 source domains + ADLOT
           • Demographics, biomarkers (20+ flags), comorbidities
           • Charlson CCI with Quan 2011 weights
           • 7 internal iterations, 1 QC fix cycle

Wave 3 [parallel]:  ADRS + ADAE (depend on ADSL)
    ├─ ADRS: RECIST BOR with confirmed response (≥28-day interval)
    └─ ADAE: AE-HO linkage, TRTEMFL derivation

Wave 4 [serial]:    ADTTE (depends on ADSL + ADRS)
    └─ ADTTE: PFS/OS/DOR with censoring rules
```

**Quality control process:**
1. Programmer implements dataset following 8-step workflow
2. Reviewer performs independent QC (BLOCKING/WARNING/NOTE findings)
3. If FAIL: programmer fixes issues (max 2 cycles)
4. If issues persist: escalate to user (0 escalations in this run)

**QC Results (Enhanced Workflow — 2026-03-29):**

| Dataset | Initial QC | Fix Cycles | Final Verdict | Key Issues |
|---------|------------|------------|---------------|------------|
| ADBS | PASS | 0 | PASS | First-run success |
| ADRS | PASS | 0 | PASS | BOR logic correct on first attempt |
| ADAE | PASS | 0 | PASS | AE-HO linkage working |
| ADTTE | PASS | 0 | PASS | Censoring rules correct |
| ADSL | PASS | 0 | PASS | Biomarker helper function prevented copy-paste pattern |
| ADLOT | FAIL | 1 | PASS | Fixed LoT algorithm (path corrections) |

**First-pass QC rate:** 83% (5 of 6 datasets) | **Escalations to user:** 0

> Baseline run (2026-03-27) achieved 67% first-pass (ADLOT + ADSL both needed fix cycles). The enhanced workflow with complexity detection and dataset splitting reduced ADSL internal iterations from 7 to 1 and improved the first-pass rate by +16%.
